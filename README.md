# AI Native 人才简历分析看板

基于 AI Native 人才标准构建的候选人评估系统，帮助招聘团队快速识别和评估 AI Native 人才。

## 🎯 功能特性

### 1. 候选人管理
- 📤 简历上传（支持 PDF、DOCX、TXT 格式）
- 📋 候选人信息录入
- 🔍 候选人搜索和筛选
- 🗑️ 候选人删除

### 2. AI Native 六维评分
根据文档定义的六个核心维度进行评估：

| 维度 | 权重 | 说明 |
|------|------|------|
| AI 优先思维 | 1.0 | 接到任务后，第一反应是"AI可以怎么帮我解" |
| 深度协作能力 | 1.0 | 将AI当成思维伙伴，能进行多轮深度协作 |
| 问题重构能力 | 1.0 | 用AI重新定义"这个问题应该是什么" |
| 学习模式 | 0.8 | "用中学、边跑边造"，用AI快速进入新领域 |
| 迭代敏捷度 | 0.8 | 每天用AI的习惯在持续进化 |
| 边界认知 | 0.8 | 清楚AI的能力边界，能精准指出不靠谱之处 |

### 3. L0-L3 等级自动评定
- **L0 - 偶尔使用者** (0-40分): 仅停留在偶尔使用AI工具
- **L1 - 工具使用者** (40-60分): 能熟练使用AI工具提效
- **L2 - 深度协作者** (60-80分): 将AI作为默认操作系统
- **L3 - 模式重构者** (80-100分): 能系统性重构工作流程

### 4. 候选人对比
- 📊 综合对比表格
- 🕸️ 六维能力雷达图
- 📝 详细评估对比

### 5. 快速判断清单
面试结束后的5个快速判断问题：
1. 候选人第一反应是"AI"而不是"人"？
2. 候选人是否拥有"自己的方法论"？
3. 候选人是否改造过自己的工作流？
4. 候选人对AI的吐槽是否具体？
5. 候选人有没有"AI焦虑"？

## 🚀 快速开始

### 方式一：使用启动脚本（推荐）

1. 双击运行 `启动看板.bat`
2. 等待依赖安装完成
3. 浏览器将自动打开看板页面

### 方式二：命令行启动

```bash
# 1. 进入项目目录
cd ai_native_resume_analyzer

# 2. 安装依赖
pip install -r requirements.txt

# 3. 启动看板
streamlit run app.py
```

### 方式三：Python 直接运行

```bash
python -m streamlit run app.py
```

## 📖 使用指南

### 添加新候选人评估

1. 点击左侧菜单「➕ 新增评估」
2. 填写候选人基本信息（姓名、职位为必填）
3. 上传简历文件（可选）
4. 根据六个维度进行评分（0-100分）
   - 每个维度都有详细的定义和关键信号参考
   - 可以添加评分说明记录具体表现
5. 完成快速判断清单
6. 填写综合评价
7. 点击「💾 保存评估」

### 查看候选人列表

1. 点击左侧菜单「📋 候选人列表」
2. 查看所有候选人的统计概览
3. 使用搜索框搜索候选人姓名
4. 使用等级筛选过滤候选人
5. 点击「查看详情」查看完整评估报告

### 候选人对比

1. 点击左侧菜单「📊 候选人对比」
2. 选择要对比的候选人（2-5人）
3. 查看综合对比表格
4. 查看六维能力雷达图
5. 展开查看详细评估

### 查看评分标准

点击左侧菜单「📖 评分标准」查看：
- 各等级定义和分数范围
- 六维评分标准的详细说明
- 快速判断清单的使用方法

## 📁 项目结构

```
ai_native_resume_analyzer/
├── app.py                 # 主应用程序
├── requirements.txt       # Python 依赖
├── 启动看板.bat          # Windows 启动脚本
├── README.md             # 使用说明
└── data/                 # 数据存储目录
    ├── candidates.json   # 候选人数据
    └── resumes/          # 简历文件存储
```

## 💾 数据存储

- 候选人数据保存在 `data/candidates.json`
- 上传的简历文件保存在 `data/resumes/`
- 数据采用 JSON 格式，便于备份和迁移

## 🔧 系统要求

- Python 3.8+
- 现代浏览器（Chrome、Firefox、Edge 等）
- 无需互联网连接（本地运行）

## 📝 评分建议

### 简历筛选重点
- ✅ 看具体场景 + 改变方式 + 产出的描述
- ✅ 寻找"用AI重建了..."、"自动化的...流程"等信号
- ✅ 注意候选人搭建的提示词库、AI工作流
- ⚠️ 警惕堆砌工具名称但没有具体使用说明的简历

### 面试问题参考
1. "如果你明天被要求独立负责一个完全不懂的行业调研，你会前半小时做什么？"
2. "你最近一次用AI解决一个不是上级指派的问题，是什么事？"
3. "请描述一个你用AI完成的最复杂的工作流"
4. "举个你发现AI搞砸了的例子，你怎么发现的，又是怎么修的？"
5. "你现在每天用AI的习惯，和半年前有什么不一样？"
